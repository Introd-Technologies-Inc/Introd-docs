## What users can do

- search for a specific person or role
- inspect the strongest warm path
- compare connectors before making an ask
- move into an introduction request when the route is real
- save useful searches and import lists into a collection workflow

## Search People

People search combines synced contacts, graph intelligence, and external enrichment signals. Results should prioritize trustworthy routes over raw directory coverage.

Use filters to narrow by connection degree, relationship strength, current company, role, and location. Good filters should reduce noise without hiding why a result was ranked.

## View Paths

For a useful result, the UI should make it obvious:

- who the target is
- who the connector is
- why the path is ranked
- whether the route is fresh enough to use

## Request Introductions

When a path looks strong, the user should be able to move directly into the intro compose flow with the target and connector already selected.

## Relationship Insights

The surrounding context matters as much as the name in the result row. Good people pages should surface trust, freshness, evidence, and next-step guidance instead of only titles and companies.
