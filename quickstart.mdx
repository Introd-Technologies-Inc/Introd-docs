## Boot order

Start the stack in this order so the product surfaces can find each other cleanly:

1. API service
2. dashboard
3. extension build
4. docs preview
5. optional graph-intel service for TrustRank and pathfinding work

## API service

Install dependencies and run the backend from `introd-api`:

```bash
npm ci
cp .env.example .env
npm run dev
```

Use non-production credentials locally. Keep `DISABLE_VITE=true` for API-only runs so the backend does not attempt to serve the retired bundled frontend.

## Run each surface

<Tabs>
  <Tab title="API">

  ```bash
  cd introd-api
  npm ci
  cp .env.example .env
  npm run dev
  ```

  The API defaults to `http://localhost:5000` and exposes the main health probes used by the dashboard and docs examples.
  </Tab>
  <Tab title="Dashboard">

  ```bash
  cd introd-dashboard
  npm ci
  cp .env.example .env
  npm run dev
  ```

  Set `INTROD_API_URL` to the local API, typically `http://localhost:5000`.
  </Tab>
  <Tab title="Extension">

  ```bash
  cd introd-extension
  npm ci
  npm run typecheck
  npm run build
  ```

  Load `dist/` as an unpacked Chrome extension and keep a signed-in LinkedIn tab open while testing sync.
  </Tab>
  <Tab title="Docs">

  ```bash
  cd introd-docs
  npx mintlify dev
  ```

  Mintlify will serve the docs locally and validate navigation, MDX, and links as you iterate.
  </Tab>
</Tabs>

## Health checks

The main readiness endpoints are:

```http
GET /health/live
GET /health/ready
GET /health/status
```

## Graph intelligence service

Run the Python TrustRank and pathfinding service separately:

```bash
cd graph-intel
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

The graph service powers TrustRank, pathfinding, and network search. Keep all graph queries tenant-scoped and evidence-backed.

## Validation pass

Before shipping backend changes:

```bash
npm run check
npm run graph-intel:test
```

Before shipping docs changes:

```bash
npx mintlify validate
```
