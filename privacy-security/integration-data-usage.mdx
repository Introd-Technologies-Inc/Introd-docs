---
title: "Integration Data Usage"
description: "What LinkedIn, Google, and Microsoft data Introd uses and why."
---

## LinkedIn

LinkedIn is used to:

- detect whether the user is signed in locally
- import visible first-degree connections from the connections page
- read profile context when the user actively opens a profile page

Introd should not claim hidden scraping or full background coverage beyond those visible, user-opened flows.

## Google

Google Workspace data is used to add:

- Gmail communication metadata
- Google Calendar meeting metadata
- Google Contacts relationship context

The value of Google in Introd is not inbox search for its own sake. It is relationship evidence that strengthens or weakens path confidence.

## Microsoft

Microsoft 365 data is used to add:

- Outlook email metadata
- Outlook calendar meeting metadata

As with Google, the key product value is relationship evidence rather than generic productivity sync.
