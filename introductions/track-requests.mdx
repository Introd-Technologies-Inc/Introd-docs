---
title: "Track Requests"
description: "Follow intro progress, bottlenecks, and connector responsiveness over time."
---

Tracking is how Introd learns whether it is producing access or just motion.

## What tracking should show

- current request status
- how long the request has been sitting
- whether the connector responded
- whether the target responded
- what next action should happen

## Tabs and filters

The dashboard intro list already supports:

- `all`
- `active`
- `in_progress`
- `completed`
- `declined`

Search can also match:

- target name
- target company
- connector name
- subject
- message
- status

## What tracking should reveal

- request status and age
- whether the connector has replied
- whether the target has replied
- where a request is blocked
- which next action should happen

## Useful status ladder

1. pending
2. approved
3. in progress
4. replied
5. meeting
6. completed
7. declined or cancelled

## Why it matters

A great intro request that disappears into a black box teaches the system nothing. Tracking is what turns a single request into future connector intelligence.
